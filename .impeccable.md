## Design Context

### Users
Nomi is a private, single-user assistant for its owner. The primary usage moments are late-night planning, checking grades or assignments, asking general questions with web search when needed, and getting help with coding tasks. The current technical focus areas include Next.js, Supabase, Neon, Clerk, Cloudflare, DigitalOcean, Postgres, Django, and the Vercel AI SDK.

The interface should optimize for solo use, repeat visits, and low-friction conversation. It should feel like a personal workspace rather than a public chatbot product or team collaboration tool.

### Brand Personality
The brand personality is private, calm, and approachable.

Emotionally, the product should create a sense of calm, momentum, delight, and friendliness. It should feel trustworthy and personal without becoming cutesy, childish, or overly branded. "Private" is a core product characteristic and should be visible in the tone and interaction model.

### Aesthetic Direction
Nomi should be dark-first, because the primary usage context is late-night planning and solo work. It should still support a light theme toggle, but dark mode is the default visual posture.

The visual system should use the provided warm coral/rust palette and existing token structure as the foundation. Blue-purple gradients, generic AI styling, and obvious chatbot-template aesthetics are explicitly out. The interface can borrow interaction cues from ChatGPT's chat input patterns, but it should not visually imitate ChatGPT or collapse into generic "AI app" design.

Typography should avoid Inter-heavy defaults. The current font direction of Plus Jakarta Sans, Lora, and IBM Plex Mono is acceptable as a baseline unless a later brand pass intentionally replaces it.

### Design Principles
1. Dark-first, light-available. Design for late-night use first, while preserving a polished light theme toggle.
2. Private by default. Every major surface should feel personal, owner-scoped, and quietly secure rather than public or social.
3. Calm momentum. The UI should help the user move forward without noise, clutter, or friction.
4. Warm precision. Use the provided coral/rust palette and soft neutrals to feel approachable without drifting into playful or childish.
5. No AI slop. Avoid blue/purple gradients, generic dashboard tropes, overly familiar chatbot shells, and default Inter-led aesthetics.
